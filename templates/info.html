How SuperX Works

A deep dive into the architecture and technology that powers this project.

Overview Core Concept

SuperX is a local, privacy-first AI image upscaler. It takes a low-resolution image and produces a version that is 4× larger in both width and height (16× the total pixel count), with sharp, realistic detail generated by a neural network.

Unlike cloud-based services, SuperX runs entirely on your own machine. Your images never leave your device — they are processed by a local Python server, enhanced by an AI model running through ONNX Runtime, and served back to your browser. No API keys, no internet connection required after setup, no data collection.

A 500×500 image becomes 2000×2000. A 1920×1080 image becomes 7680×4320 (8K resolution).

Processing Pipeline — Step by Step Detailed

Here is exactly what happens from the moment you click "Process Image" to the moment you see the enhanced result:

1

Upload & Validate

Your browser sends the image to the Flask server via an AJAX POST request. The server validates the file type (PNG/JPG), saves it to a temporary directory, and assigns a unique task ID.

2

Background Thread

A Python background thread is spawned so the server can immediately respond with the task ID. Your browser begins polling /status/<task_id> every 500ms to get real-time progress updates.

3

Image Decoding

The background thread opens the image using Pillow (PIL), converts it to RGB format, and reads its pixel dimensions to calculate how many tiles are needed.

4

Tile Extraction

The image is sliced into overlapping 100×100 pixel tiles. Overlap ensures seamless stitching. Edge tiles are padded to maintain consistent input dimensions for the model.

5

ESRGAN Inference

Each tile is normalized to a float32 tensor, reshaped to [1, 3, H, W], and fed into the ONNX Runtime session running the ESRGAN model. The model outputs a 4× upscaled tile.

6

Tile Stitching

All enhanced tiles are placed onto a blank canvas at their corresponding 4× coordinates. Overlapping regions are blended using alpha compositing to eliminate visible seams.

7

Save & Serve

The final stitched image is saved as a PNG to the results directory. The task status is set to "completed" and the browser is redirected to the result page.

What is ESRGAN?

ESRGAN stands for Enhanced Super-Resolution Generative Adversarial Network. It was published by Xintao Wang et al. in 2018 and won first place in the PIRM2018 Challenge on perceptual super-resolution.

It consists of two neural networks trained against each other:

  • Generator: Takes a low-res patch and tries to produce a convincing high-res version with realistic textures.
  • Discriminator: Tries to distinguish between the generator's output and real high-res photos.

This adversarial training forces the generator to produce outputs that are visually indistinguishable from real photographs, rather than blurry averaged results from simpler methods.

ESRGAN vs Traditional Upscaling
Method Quality Speed
Nearest Neighbor Pixelated, blocky Instant
Bilinear Smooth but blurry Instant
Bicubic Slightly sharper, still blurry Instant
Lanczos Good edges, soft textures Fast
ESRGAN (SuperX) Sharp, realistic detail ~10–30s

ESRGAN doesn't just resize — it generates new detail that never existed in the original image.

Why Tile-Based Processing? Memory

Running a neural network on an entire image at once requires loading every pixel into GPU or CPU memory simultaneously. For a 1920×1080 image, the ESRGAN model would need to process a tensor of shape [1, 3, 1080, 1920] — that's ~24 million float32 values just for the input, and the intermediate layers require even more.

On most consumer hardware, this would instantly crash with an Out of Memory error. SuperX solves this by breaking the image into small, manageable tiles:

  • Tile size: 100×100 pixels (configurable)
  • Overlap: 10 pixels on each side to prevent visible seams at tile boundaries
  • Blending: Overlapping regions are averaged during stitching for seamless output
  • Memory usage: Each tile only needs ~120KB of memory, so even a 4GB RAM machine can process 8K images
Technology Stack Under the Hood
Python 3 + Flask
Lightweight web server that handles routing, file uploads, and serves the HTML frontend. Debug mode enables hot-reloading during development.
ONNX Runtime
High-performance inference engine from Microsoft. Runs the exported ESRGAN model with optimized C++ backends. Supports CPU and optional GPU acceleration.
Pillow (PIL)
Image manipulation library used for reading, resizing, converting, and saving images. Handles the tile extraction and stitching operations.
NumPy
Performs the numerical tensor transformations — normalizing pixel values from [0, 255] to [0.0, 1.0] float32 and reshaping arrays for model input.
Threading
Python's threading module runs the AI inference in a background thread so the web server remains responsive and can serve status polling requests.
Vanilla JS + CSS
No React, no Vue, no frameworks. The frontend uses plain JavaScript Fetch API for AJAX uploads and CSS variables for the design system.
